# ComamosYA
Proyecto Práctico
🍔 ComamosYa

Dejen de discutir sobre dónde comer. Desliza, haz match y disfruta.

Este repositorio documenta la fase inicial de conceptualización y prototipado de ComamosYa, una aplicación diseñada para resolver el clásico problema de decidir dónde comer en grupo.

📖 1. Explicación del Negocio

El Problema

Muchas veces, cuando un grupo de personas intenta decidir dónde comer, se generan discusiones o se pierde mucho tiempo comparando opciones. La "paradoja de la elección" dicta que mientras más alternativas existen, más difícil resulta tomar una decisión. Los grupos suelen enfrentarse a:

No ponerse de acuerdo sobre qué restaurante elegir.

Pasar demasiado tiempo revisando opciones (en mapas o apps de delivery).

Generar pequeñas discusiones, frustraciones o que alguien termine cediendo a regañadientes.

La Solución

ComamosYa busca facilitar la toma de decisiones grupales mediante un sistema de "swiping" (deslizar). Cada participante indica sus preferencias deslizando opciones de restaurantes cercanos. El sistema identifica las alternativas en las que todos coinciden (Match), evitando que alguno quede insatisfecho y gamificando el proceso de decisión.

⚙️ 3. Justificación Metodológica

Para el desarrollo ágil de este proyecto se utiliza una combinación estratégica de Lean Startup y Scrum:

Lean Startup: Nos permite validar rápidamente si la solución (el swiping para grupos) realmente resuelve el problema de los usuarios antes de realizar grandes inversiones de tiempo y código. La prioridad es el aprendizaje validado.

Scrum: Facilita la organización del trabajo técnico y de diseño mediante tareas divididas en períodos muy cortos (sprints), permitiendo realizar entregas y mejoras continuas.

Justificación: La combinación de ambas metodologías ayuda a construir el producto de manera ordenada, minimizando riesgos y desperdicios, mientras se obtiene retroalimentación constante de los usuarios (Early Adopters).
